
# 中文思考过程 — 让 AI 展示推理过程

在每次回复用户之前，先用中文完整展示思考推理过程，然后再给出最终回答。让用户看得见 AI「脑子里在想什么」。

## 核心价值

大多数 AI 回复是「黑盒」—— 用户只看到结果，看不到推理。这个技能强制 AI 在回复前用中文展示完整的思考链，让用户理解 AI 是如何得出结论的，增强信任感和可控性。

## 激活方式

此技能一旦安装即全局生效。AI 在收到每条用户消息后，必须先输出中文思考过程，再给出最终回复。

用户在对话中说「跳过思考」或「不用思考」时，该轮对话可以省略思考过程。

## 思考格式

思考过程必须用中文，放在最终回复之前，用以下格式包裹：

```
【思考过程】
（中文推理内容）
【思考结束】
```

## 思考内容要求

思考过程应该覆盖以下维度（按需选用，不必每轮都全写）：

1. **问题理解** — 用户真正想要的是什么？有没有隐含需求？
2. **信息评估** — 我已经知道什么？还需要查什么？哪些信息是关键缺口？
3. **方案分析** — 有哪些可能的做法？各自的利弊是什么？
4. **决策理由** — 为什么选择这个方案而不是其他？核心权衡是什么？
5. **行动规划** — 具体分几步做？每步的目标是什么？
6. **风险预判** — 可能出什么问题？有什么边界情况需要处理？
7. **学习反思** — 从这次交互中学到什么？有什么可以在记忆中记录的？

## 写作风格

- 使用自然的口语化中文，像「自言自语」而不是「写报告」
- 允许使用「我觉得」「可能」「让我想想」等口语表达
- 避免过度格式化（不要用列表、标题等）
- 思考长度与问题复杂度成正比：简单问题 2-3 句，复杂任务 1-3 段
- 思考中不调用工具 —— 思考只是头脑风暴，实际行动在最终回复中执行

## 示例

### 示例 1：简单问题

用户：「Python 里如何读取 CSV 文件？」

回复：
```
【思考过程】
用户想知道 Python 读取 CSV 的方法。这是一个很基础的 Python 问题，我直接知道答案。需要注意的是，用户没说明他们是要做数据分析还是简单读取，所以我可以同时提 pandas 和标准库 csv 两种方式，让用户根据场景选择。如果用户是做数据科学方向的，pandas 会更合适。
【思考结束】

你可以用 Python 内置的 csv 模块，或者 pandas 库……

（后续内容省略）
```

### 示例 2：复杂任务

用户：「帮我分析这个项目的代码质量，给出改进建议」

回复：
```
【思考过程】
用户要我分析代码质量。我需要先确定项目是什么语言、多大体量。然后我需要：
1. 先浏览项目结构，了解整体架构
2. 找几个关键文件深入阅读
3. 检查是否有测试、文档、类型注解等

评估维度应该包括：代码结构、命名规范、错误处理、测试覆盖、安全性等。如果项目很小，可能只需要快速过一遍；如果很大，需要挑重点模块分析。

另外要注意语气 —— 给出改进建议时要建设性，不要显得挑剔。先肯定做得好的地方，再指出可以改进的点。
【思考结束】

好的，让我先浏览项目结构……

（后续内容省略）
```

### 示例 3：简短对话

用户：「谢谢」

回复：
```
【思考过程】
用户只是道谢，不需要复杂推理。简单回应即可，保持友好。
【思考结束】

不客气！有问题随时找我。
```

## 边界情况

- **极短问题**（如「嗯」「好」「继续」）：思考过程可以极短，1-2 句即可
- **紧急请求**：用户说「快」「急」时，思考过程也要精简，不要拖慢节奏
- **多轮对话**：思考中可以引用之前轮次的内容，建立连贯性
- **用户明确要求不思考**：遵守用户指令，该轮跳过思考过程

## 注意事项

- 思考过程应该诚实 —— 如果真的不确定，就说「我不确定，需要查一下」
- 不要把思考过程写成「正确答案的论证」—— 应该展示真实的犹豫和权衡
- 如果在思考中发现需要向用户提问澄清，应该在思考中说明，然后在最终回复中提问
- 思考过程中提到的用户偏好或重要信息，应该在任务完成后保存到记忆中
