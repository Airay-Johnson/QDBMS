# QDBMS 项目改造方案

> 从 JavaFX 桌面应用 + openGauss → Spring Boot + Vue 3 + MySQL 全栈 Web 应用

---

## 一、现状分析

| 维度 | 当前 | 目标 |
|------|------|------|
| 架构 | JavaFX 桌面应用 | B/S Web 应用 |
| 后端框架 | 无框架，手写 Servlet/JavaFX | Spring Boot 3.x |
| 数据库 | openGauss (PostgreSQL系) | MySQL 8.0 |
| ORM | 手写 JDBC + DAO | MyBatis Plus |
| 前端 | JavaFX FXML | Vue 3 + Element Plus |
| 认证 | 密码+salt，Session | JWT + Spring Security |
| 权限 | 手写 RBAC | Spring Security RBAC |
| API | 无（桌面直连DB） | RESTful API |
| 部署 | 客户端安装 | Docker + Nginx |

**现有业务模块（均可复用）：**

- 用户管理：注册/登录，admin/user 角色
- 问卷管理：创建/编辑/发布/关闭/删除
- 题库管理：题型支持单选/多选/文本
- 受访者管理：导入/分组/答卷记录
- 数据收集：提交答卷
- 数据分析：统计 + 文本分析
- 报表导出：Excel (Apache POI)
- 权限管理：RBAC (角色-权限-用户)

**现有数据模型（13个实体）：**
User, Questionnaire, Question, Option, Answer, Respondent, Response, ResponseData, Role, Permission, Log, SurveyResponse, UserProfile

## 二、数据库改造

### 2.1 迁移策略

openGauss → MySQL 映射：

| openGauss | MySQL |
|-----------|-------|
| SERIAL/BIGSERIAL | BIGINT AUTO_INCREMENT |
| VARCHAR | VARCHAR |
| TIMESTAMP | DATETIME |
| BOOLEAN | TINYINT(1) |
| JSON | JSON (MySQL 8.0 原生支持) |
| TEXT | TEXT |

### 2.2 精简后的表设计（10张核心表）

```sql
-- 1. 用户表
CREATE TABLE user (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    email VARCHAR(100),
    is_active TINYINT(1) DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 2. 角色表
CREATE TABLE role (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    role_name VARCHAR(50) NOT NULL UNIQUE COMMENT 'ADMIN, USER, ANALYST',
    description VARCHAR(255)
);

-- 3. 权限表
CREATE TABLE permission (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    perm_name VARCHAR(100) NOT NULL UNIQUE COMMENT 'survey:create, survey:delete',
    description VARCHAR(255)
);

-- 4. 用户-角色关联
CREATE TABLE user_role (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    role_id BIGINT NOT NULL,
    FOREIGN KEY (user_id) REFERENCES user(id),
    FOREIGN KEY (role_id) REFERENCES role(id),
    UNIQUE(user_id, role_id)
);

-- 5. 角色-权限关联
CREATE TABLE role_permission (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    role_id BIGINT NOT NULL,
    perm_id BIGINT NOT NULL,
    FOREIGN KEY (role_id) REFERENCES role(id),
    FOREIGN KEY (perm_id) REFERENCES permission(id),
    UNIQUE(role_id, perm_id)
);

-- 6. 问卷表
CREATE TABLE questionnaire (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    creator_id BIGINT NOT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    status TINYINT(1) DEFAULT 0 COMMENT '0=未发布 1=已发布 2=已关闭',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    published_at DATETIME,
    closed_at DATETIME,
    FOREIGN KEY (creator_id) REFERENCES user(id)
);

-- 7. 问题表
CREATE TABLE question (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    questionnaire_id BIGINT NOT NULL,
    question_text TEXT NOT NULL,
    type VARCHAR(20) NOT NULL COMMENT 'single, multiple, text',
    options JSON COMMENT '选项JSON: [{"label":"A","value":"选项A"}]',
    sequence_number INT DEFAULT 0,
    is_required TINYINT(1) DEFAULT 1,
    FOREIGN KEY (questionnaire_id) REFERENCES questionnaire(id) ON DELETE CASCADE
);

-- 8. 受访者表
CREATE TABLE respondent (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    questionnaire_id BIGINT NOT NULL,
    name VARCHAR(100),
    email VARCHAR(100),
    phone VARCHAR(20),
    group_name VARCHAR(50),
    FOREIGN KEY (questionnaire_id) REFERENCES questionnaire(id) ON DELETE CASCADE
);

-- 9. 答卷表（一次提交一份）
CREATE TABLE response (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    questionnaire_id BIGINT NOT NULL,
    respondent_id BIGINT,
    submitted_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    ip_address VARCHAR(45),
    FOREIGN KEY (questionnaire_id) REFERENCES questionnaire(id),
    FOREIGN KEY (respondent_id) REFERENCES respondent(id)
);

-- 10. 答案明细表
CREATE TABLE answer (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    response_id BIGINT NOT NULL,
    question_id BIGINT NOT NULL,
    answer_text TEXT,
    answer_options JSON COMMENT '多选题选项',
    FOREIGN KEY (response_id) REFERENCES response(id) ON DELETE CASCADE,
    FOREIGN KEY (question_id) REFERENCES question(id)
);
```

### 2.3 数据迁移脚本

使用 Python/Shell 脚本从 openGauss 导出 → 转换 → 导入 MySQL。保留原有业务数据。

### 3.4 关键代码示例

**统一响应格式：**
```java
@Data
public class Result<T> {
    private int code;
    private String message;
    private T data;
    
    public static <T> Result<T> ok(T data) {
        Result<T> r = new Result<>();
        r.code = 200;
        r.message = "success";
        r.data = data;
        return r;
    }
}
```

**JWT 认证过滤器：**
```java
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest request,
            HttpServletResponse response, FilterChain chain) {
        String token = extractToken(request);
        if (token != null && jwtProvider.validate(token)) {
            String username = jwtProvider.getUsername(token);
            UserDetails userDetails = userDetailsService.loadUserByUsername(username);
            SecurityContextHolder.getContext()
                .setAuthentication(new UsernamePasswordAuthenticationToken(
                    userDetails, null, userDetails.getAuthorities()));
        }
        chain.doFilter(request, response);
    }
}
```

### 4.4 Axios 拦截器

```javascript
// JWT Token 自动注入 + 401 自动跳转登录
request.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

request.interceptors.response.use(
  response => response.data,
  error => {
    if (error.response?.status === 401) {
      localStorage.removeItem('token')
      router.push('/login')
    }
    return Promise.reject(error)
  }
)
```

## 六、与原项目的对比

| 维度 | 旧（JavaFX） | 新（Spring Boot + Vue） |
|------|-------------|------------------------|
| 用户使用方式 | 安装 .exe 客户端 | 浏览器打开 URL |
| 受访者填问卷 | 不可能（桌面软件） | 分享链接即可 |
| 多用户协作 | 不支持 | 天然支持 |
| 移动端 | 不支持 | 响应式设计 |
| API | 无 | RESTful，可被第三方调用 |
| 数据库连接 | 直连（密码暴露） | 后端代理，安全 |
| 维护成本 | 客户端更新 | 服务端热更新 |
| 面试展示 | 一般 | 企业级技术栈，亮眼 |
